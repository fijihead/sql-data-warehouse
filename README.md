# SQL Data Warehouse

## Project Overview

This project will provide an example of how a SQL data warehouse can be designed and built with ETL processing.


## Objective
- Actionable insights to all stakeholders
- Facilitate analysis of customer behavior, product performance, and sales trends
- Use star schema modeling


## Scope
- The staging and Transformation of Raw Data
- Development of reusable analytics views for reporting
- Scalable design to accommodate future growth with new datasets


## Data Sources
- Sample transactional data including customer, product, and sales information


## Data Quality & Integration
- Pre-processed input data in the ETL process
- Guaranteed referential integrity between the fact and the dimension tables
- Managed missing and duplicated values correctly


## Analytics & Reporting
- Prepared SQL views for analytical workflows and dashboards
- Supports insights on:
 - Customer behavioral patterns
 - Product sales and performance
 - Revenue & trend analysis

 
 ## Skills Demonstrated in this project 
- SQL and Data Warehousing Topics
- ETL Processes and Data Transformations
- Star Scheme and Dimensional Modeling
- Optimization of queries for analytics
